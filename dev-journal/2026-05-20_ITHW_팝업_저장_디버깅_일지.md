## 발견한 문제 목록

### ✅ 수정 완료

**1. ASSET_NO 변수 누락**
- 문제: `let ASSET_NO` 선언이 없어서 `dataLoadForm()`에서 조회 조건 없이 날아감
- 수정: `let ASSET_NO = "${fn:escapeXml(param.asset_no)}";` 추가
- 결과: 팝업에서 데이터 로드 됨

---

### ❌ 수정 필요

**2. form_ids 컬럼명 오류**

| 잘못된 이름 | 실제 컬럼명 | 항목 |
|---|---|---|
| `use_server` | `asset_loc` | IDC 위치 |
| `virtual_yn` | `vtl_yn` | 가상화 여부 |
| `cpny_url` (SW) | `cpny_domain` | 업체 URL |

**3. rmk vs reqm_rmk ID 불일치**
- JSP: `<textarea id="reqm_rmk">`
- form_ids: `'rmk'` 사용 중
- 수정: form_ids의 `'rmk'` → `'reqm_rmk'` 변경

**4. VTL_YN 공통코드 미등록**
- `ENG_CODE_DETAIL` 테이블에 `MSTR_CD = 'VTL_YN'` 데이터 없음
- 결과: 가상화 여부 드롭다운에 Y/N 옵션 없음
- 수정 INSERT 쿼리:

```sql
INSERT INTO ENG_CODE_DETAIL 
(MSTR_CD, DTL_CD, DTL_CD_NM, DTL_CD_TYPE, DTL_CD_DESC, ORD, USE_YN, REG_EMP_NO, REG_DT)
VALUES ('VTL_YN', 'Y', 'Y', 'HW', NULL, 1, 'Y', 2104428, GETDATE());

INSERT INTO ENG_CODE_DETAIL 
(MSTR_CD, DTL_CD, DTL_CD_NM, DTL_CD_TYPE, DTL_CD_DESC, ORD, USE_YN, REG_EMP_NO, REG_DT)
VALUES ('VTL_YN', 'N', 'N', 'HW', NULL, 2, 'Y', 2104428, GETDATE());
```

**5. settingCtrt 오류**
- `validateNumber('exec_amnt')` 호출하는데 JSP에 `#exec_amnt` 없음
- JSP 실제 필드명: `actual_amnt`
- 수정: `validateNumber('actual_amnt')` 로 변경

**6. saveItAsset dataType 주석**
- `// ,dataType : 'json'` 주석 처리됨
- result 파싱 문제 가능성
