# 2026-05-20 Oracle DB P6 스키마 백업 준비 작업 일지

## 작업 목적

- `.169 → .184` P6 DB 이전 작업 연속
- P6 스키마 4개 백업 전 Oracle 환경 설정

---

## 작업 내역

### 1. Oracle 접속 방법 확인

| 시도 | 결과 |
|---|---|
| `sqlplus / as sysdba` | ✅ 성공 (로컬 sysdba) |
| `CONNECT system/admin123@XEPDB1` | ❌ ORA-01017 비밀번호 오류 |
| `CONNECT system/admin@123@XEPDB1` | ❌ ORA-12154 TNS 분석 불가 |
| `CONNECT system/"admin@123"@XEPDB1` | ✅ 성공 |

> **핵심**: 특수문자(`@`) 포함 비밀번호는 **큰따옴표**로 감싸야 함

---

### 2. 백업 디렉토리 생성

```sql
CREATE OR REPLACE DIRECTORY BACKUP_DIR AS 'D:\BACKUP';
-- 결과: 디렉토리가 생성되었습니다.
```

---

### 3. expdp 전체 백업 시도 → 실패

```cmd
expdp system/\"admin@123\"@XEPDB1 full=y directory=BACKUP_DIR dumpfile=xedump.dmp logfile=xedump.log
```

**에러 내용**
- ORA-22303: SYS.DPAPI_JOB_RECORD 유형 없음
- ORA-04036: PGA_AGGREGATE_LIMIT 초과
- ORA-06508: SYS.DBMS_DATAPUMP 찾을 수 없음

---

### 4. PGA 메모리 제한 확인 및 해제

```sql
-- 현재 설정 확인
SHOW PARAMETER PGA_AGGREGATE_LIMIT;
-- 결과: 2880M

-- 제한 해제 (0 = 무제한)
ALTER SYSTEM SET PGA_AGGREGATE_LIMIT = 0 SCOPE=BOTH;
-- 결과: 시스템이 변경되었습니다.
```

---

### 5. PGA 사용량 확인

```sql
SELECT round(pga_used_mem/1024/1024,1) as 사용중MB
FROM v$process
ORDER BY pga_used_mem DESC
FETCH FIRST 5 ROWS ONLY;
```

| 순위 | 사용중(MB) |
|---|---|
| 1 | 864.5 |
| 2 | 164.9 |
| 3 | 103.6 |
| 4 | 103.5 |
| 5 | 103.5 |
